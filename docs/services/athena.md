# Athena

**Protocol:** JSON 1.1
**Endpoint:** `http://localhost:4566/`

Floci emulates Amazon Athena by using **DuckDB** as the underlying SQL engine. It allows you to run SQL queries against data stored in S3 using the Glue Data Catalog.

## Supported Actions

| Action | Description |
|---|---|
| `StartQueryExecution` | Initiates an asynchronous SQL query |
| `GetQueryExecution` | Checks the status of a query (QUEUED, RUNNING, SUCCEEDED, FAILED) |
| `GetQueryResults` | Returns the rows generated by a successful query |
| `ListQueryExecutions` | Returns a list of past query executions |
| `StopQueryExecution` | Cancels a running query |
| `CreateWorkGroup` | Creates a new workgroup |
| `GetWorkGroup` | Returns information about a workgroup |
| `ListWorkGroups` | Lists all workgroups |

## How it works

1. **SQL Rewriting**: Floci automatically detects table names in your SQL and maps them to physical S3 locations defined in the Glue Catalog.
2. **DuckDB Engine**: Queries are executed in a background thread pool using DuckDB.
3. **Format Support**: Supports querying **Parquet** and **CSV** files stored in S3.
4. **S3 Integration**: DuckDB reads directly from Floci's emulated S3 service.

## Example

```bash
export AWS_ENDPOINT_URL=http://localhost:4566

# Start a query
QUERY_ID=$(aws athena start-query-execution \
  --query-string "SELECT count(*) FROM my_table" \
  --query-execution-context Database=my_db \
  --query 'QueryExecutionId' \
  --output text \
  --endpoint-url $AWS_ENDPOINT_URL)

# Wait for completion and get results
aws athena get-query-results --query-execution-id $QUERY_ID --endpoint-url $AWS_ENDPOINT_URL
```
